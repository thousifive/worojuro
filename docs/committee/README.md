# Worojuro feature committee archive

Every feature review — approved, conditional, or
rejected — is logged here as a markdown file named
`[YYYY-MM-DD]-[feature-slug].md`

This log is the single source of truth for why
features were or were not built. The delivery manager
references it during sprint planning. The strategist
uses it to identify patterns in what the team
consistently flags as risky.

Files in this folder are never deleted.
